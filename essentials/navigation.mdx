Het navigatiemenu is de lijst met links op elke website.

Je zult `docs.json` waarschijnlijk elke keer bijwerken wanneer je een nieuwe pagina toevoegt. Pagina's verschijnen niet automatisch.

## Navigatiesyntax

Onze navigatiesyntax is recursief, wat betekent dat je geneste navigatiegroepen kunt maken. Je hoeft `.mdx` niet op te nemen in paginanamen.

<CodeGroup>

```json Reguliere navigatie
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Aan de slag",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Geneste navigatie
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Aan de slag",
          "pages": [
            "quickstart",
            {
              "group": "Geneste referentiepagina's",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Mappen

Plaats je MDX-bestanden gewoon in mappen en werk de paden bij in `docs.json`.

Om bijvoorbeeld een pagina te hebben op `https://yoursite.com/your-folder/your-page` zou je een map maken genaamd `your-folder` met daarin een MDX-bestand genaamd `your-page.mdx`.

<Warning>

Je kunt `api` niet gebruiken als naam voor een map, tenzij je deze nest in een andere map. Mintlify gebruikt Next.js dat de top-level `api` map reserveert voor interne server calls. Een mapnaam zoals `api-reference` zou worden geaccepteerd.

</Warning>

```json Navigatie met map
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Groepsnaam",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Verborgen pagina's

MDX-bestanden die niet zijn opgenomen in `docs.json` verschijnen niet in de zijbalk, maar zijn toegankelijk via de zoekbalk en door direct naar ze te linken.
